Spezifikationen:

• Funktional:
- Das System soll es Nutzern ermöglichen ein Nutzerprofil anzulegen
- Nutzer können beim registrieren auswählen ob sie Studierende, Prüfende oder Dozierende sind
- Dozierende sollen alle Rechte haben die Prüfende haben (und mehr)
- Das System soll es Nutzern ermöglichen das eigene Nutzerprofil zu editieren
- Das System soll es Nutzern ermöglichen das eigene Nutzerprofil zu speichern
- Das System soll es Nutzern ermöglichen das Nutzerprofil mit zusätzlichen Daten zu erweitern
- Das System soll es Dozierenden ermöglichen Aufgaben hochzuladen
- Das System soll es Dozierenden ermöglichen Lösungen hochzuladen
- Dozierende sollen Aufgaben im nachhinein noch bearbeiten können
- Dozierende sollen Lösungen im nachhinein noch bearbeiten können
- Das System soll es Dotzierenden ermöglichen eine Aufgabe einem Themengebiet zuzuweisen
- Das System soll es Dozierenden ermöglichen eine Aufgabe einer Schwierigkeitsstufe zuzuweisen
- Das System soll es Dozierenden ermöglichen einer Aufgabe einen Zeitbedarf zuzuweisen
- Nutzer sollen vom System eine nach einegegebenen Parametern generierte Testklausur erhalten können
- Nutzer sollen wählen können ob sie Lösungen für generierte Testklausuren bekommen
- Das System soll es Prüfern ermöglichen alle Aufgaben einzusehen
- Das System soll es Prüfern ermöglichen Übungsblätter zusammenzustellen
- Das System soll es Prüfern ermöglichen einem Übungsblatt eine Musterlösung anzuhängen
- Das System soll es dem Prüfenden ermöglichen Klausuren zu erstellen
- Der Prüfende sollen vom System eine Hilfestellungen zum Klausuren erstellen bekommen

• nicht Funktional:

Technisch:
- Das System soll mit Python 3.10 entwickelt werden
- Das System soll das Django Framework nutzen
- Das System soll über einen Webbrowser erreichbar sein
- Das System muss mit Chrome Version 96 und Firefox Version 94 und neuer kompatibel sein

Qualität:
- Das System soll für keine Antwort mehr als 20 Sekunden brauchen
- Das System soll immer eine Antwort liefern (kein Fehlercode oder Crash)
- Die Website soll übersichtlich gestaltet sein

Benutzerschnittstelle:
- Das System soll als Eingabe Pdf Dateien akzeptieren
- Das System soll als Ausgabeformat Pdf benutzen
- Nutzer sollen immer auf ihr Profil zugreifen können

Rechtlich-Vertraglich:
- Das Passwort soll verschlüsselt sein (gehashed)
- Als Nutzer muss man sich authentifizieren
- Nutzern sollen nicht auf die Profile anderer Nutzer zugreifen können
- Das Nutzerprofil soll durch ein Passwort geschützt sein

Prozess:
- Deadline am 01.04.2022
- Es muss eine Abschlusspräsentation geben
